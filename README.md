## Основной файл для проверки

Главный файл репозитория: **`report.txt`**.

В нём последовательно размещены:
1. очищенная история команд, относящихся только к выполнению ДЗ;
2. содержимое `/etc/passwd`;
3. содержимое `/etc/group`;
4. результаты финального тестирования.

## Что сделано

### 1. Управление сетью
- настроен статический IP `192.168.1.100/24`;
- настроены шлюз `192.168.1.1` и DNS `8.8.8.8`;
- установлен hostname `mephi-2026.domain.local`;
- выполнена проверка связности с локальной сетью и внешним миром.

### 2. Управление программным обеспечением
- установлены пакеты `nginx`, `tcpdump`, `libcap-ng-utils`;
- RPM-пакет `tcpdump` скачан через `dnf download` и установлен через `rpm`.

### 3. Файловые системы и сервисы
- создан раздел `/dev/vdb1`;
- создана файловая система `ext4` с меткой `MEPHI_DATA`;
- настроено автомонтирование в `/data/mephi-web` через `/etc/fstab`;
- `nginx` запущен и включён в автозапуск.

### 4. Управление доступом
- создан пользователь `mephi-admin`;
- создана группа `mephi-devs`;
- на `/data/mephi-web` установлены владелец `mephi-admin:mephi-devs` и права `2775`;
- SELinux работает в режиме `Enforcing`;
- для web-каталога назначен контекст `httpd_sys_content_t`;
- для `tcpdump` настроены capabilities.

### 5. Аутентификация и тестирование
- вход `root` ограничен;
- создан файл `index.html` с персонализированной страницей;
- `curl http://localhost` и `curl http://192.168.1.100` возвращают корректный результат;
- выполнена дополнительная проверка после перезагрузки системы.
